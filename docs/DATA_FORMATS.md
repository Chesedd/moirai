# Форматы данных

Описание всех файлов в Google Drive и того, как бот интерпретирует
ввод пользователя.

## inbox.md

Append-only лог всех сообщений от пользователя. Каждая строка
самодостаточна: таймстамп, тип, содержимое.

```
[2026-04-28 14:32] TASK: купить хлеб #личное
[2026-04-28 14:33] TASK: отчёт Q1 !важно до пятницы #работа
[2026-04-28 15:10] DONE: сдал отчёт
[2026-04-28 15:45] EVENT: завтра 10:00 стоматолог
[2026-04-28 16:00] SLOT: сегодня 17-18 работа
[2026-04-28 16:01] SLOT: пн-пт 9-13 офис
[2026-04-28 17:15] NOTE: подумать над стратегией на Q2
[2026-04-28 18:30] SKIP: купить хлеб
```

### Типы записей

**TASK** — то, что нужно сделать. Опциональные атрибуты Claude разбирает
сам:

- дедлайн естественным языком — `до пятницы`, `завтра`, `асап`,
  `к концу месяца`;
- важность — флагом `!важно` или `!критично`;
- теги — через `#` (`#работа`, `#личное`, любые).

**DONE** — отметка о выполнении. Ссылается на ближайшую подходящую
`TASK` по смыслу; матч делает Claude. Писать длинно не нужно.

**EVENT** — событие в конкретный момент времени. Нет дедлайна, есть
момент наступления. Разовые.

**SLOT** — занятой интервал. Бывает разовым (`сегодня 17-18`) или
повторяющимся (`пн-пт 9-13`, `каждую среду 19-21`). Отражает реальные
временные границы дня — фиксированная работа, учёба, сон.

**NOTE** — мысль или идея, пока не задача. Попадает в недельный обзор
с вопросом «что с этим делать: превратить в TASK, в EVENT или удалить».

**SKIP** — отметка о пропуске задачи. Записывается ботом в ответ на
команду `/skip N`. По смыслу: «эту задачу сегодня не делаю,
перенеси на следующий день». В отличие от DONE, не закрывает
задачу — только отмечает дневной пропуск. Routines (`daily_plan`,
`priorities`) учитывают SKIP так:

- задача с SKIP в течение последних 24 часов исключается из
  «Сегодня» в `daily_plan_short.md`;
- но остаётся в backlog и в `priorities_short.md`;
- количество SKIP по одной задаче — это счётчик переносов,
  отображается надстрочной цифрой `²`, `³` рядом с задачей;
- задачи с тремя и более SKIP флагуются как «вечно висящие» с
  предложением удалить или переосмыслить.

`weekly_review` НЕ архивирует SKIP — это рабочая метка, не история.

### Роль бота

Бот не парсит содержимое, только определяет тип — по команде, кнопке
или простейшей эвристике (`сделал X` / `сделано X` → DONE, остальное
по умолчанию → TASK) — приписывает таймстамп и дописывает строку.
Всё смысловое разбирательство делает Claude внутри routines.

## outputs/*.md и outputs/*_short.md

Каждый артефакт routines существует в двух версиях:

- **Полная** — `daily_plan.md`, `weekly_review.md`, `priorities.md`.
  Markdown-файл свободного формата, читаем человеком, лежит в Drive
  для истории и для self-reference у самих routines (например,
  `priorities` опирается на вчерашний `daily_plan.md`).
- **Короткая** — `daily_plan_short.md`, `weekly_review_short.md`,
  `priorities_short.md`. Готовый Telegram-вид, который бот
  пересылает в чат как есть, без парсинга и переформатирования.

Routine, генерирующий артефакт, обязан писать обе версии в одном
запуске. Бот следит только за `*_short.md` файлами в `outputs/` —
полные версии не пересылаются. Если короткая версия отсутствует,
бот молчит и ждёт следующего обновления.

Это разделение уводит парсинг markdown из бота в сторону Claude,
который и так формирует контент — ему дешевле один раз отдать
короткую формулировку, чем боту парсить произвольный markdown
регулярками.

`schedule.md` — исключение: машинно-читаемый, `*_short.md` версии
не имеет. Его читает таймер бота, а не человек.

## outputs/daily_plan.md

Полная версия плана на день. Перезаписывается routine `daily_plan`
каждый день в 08:00. Бот рендерит компактную Telegram-версию на лету.

```markdown
# Пятница, 24 апреля 2026

## Таймлайн

09:00–13:00  офис (пн-пт)
13:00–17:00  свободно (4 ч)
17:00–18:00  работа
18:00–22:00  свободно (4 ч)

⏰ 10:00 — стоматолог

## В приоритете

🔴 **Отчёт Q1** — дедлайн сегодня · #работа
   рекомендую 13:00–17:00

🟡 **Позвонить подрядчику** — #работа
   короткое, подойдёт начало окна 13:00

## Сегодня же

- Купить хлеб · #личное
- Забрать посылку · #личное (переношу 2-й раз — давай сегодня?)

## Заметки

На сегодня 2 задачи с высоким приоритетом и 2 бытовых. В свободных
окнах около 4 часов активной работы — комфортно, зазор под неожиданное
есть. Обрати внимание: «забрать посылку» висит неделю — если сегодня
опять не получится, возможно стоит удалить.
```

## outputs/weekly_review.md

Обзор завершающейся недели. Включает Eisenhower-матрицу для открытых
задач, статистику выполнения, подсветку вечно висящих пунктов и
чистку старых записей в архив. Формат детально проработаем в Фазе 4.

## outputs/priorities.md

Ранжированный список всех открытых `TASK` с Eisenhower-разбивкой.
Обновляется ежедневно в 21:00. Формат детально проработаем в Фазе 4.

## outputs/schedule.md

Упрощённый список нормализованных дат для бот-таймера. Формат
строго машиночитаемый — парсится одной регуляркой, Claude ничего
«красивого» сюда не пишет.

```
# Расписание (обновлено 2026-04-24 08:00)

2026-04-24 10:00       | EVENT | стоматолог
2026-04-24 17:00-18:00 | SLOT  | работа
2026-04-25 09:00-13:00 | SLOT  | офис
2026-04-25 14:00       | EVENT | созвон с командой
```

Схема строки: `ДАТА ВРЕМЯ | ТИП | ОПИСАНИЕ`. Повторяющиеся `SLOT`
разворачиваются в конкретные даты на 7 дней вперёд. Прошедшие записи
удаляются при следующем обновлении.

## archive/YYYY-W##.md

В воскресенье в 20:00 routine `weekly_review` переносит сюда:

- `DONE` старше 7 дней;
- `EVENT` в прошлом;
- разовые `SLOT` в прошлом;
- `NOTE`, явно помеченные как неактуальные.

Inbox.md остаётся компактным. Архив routines не читают — он сохраняется
для ретроспективы «что я делал в апреле».

Купить хлеб

💬 4ч активной в окнах — норм

/done N · /skip N · /plan

Эмодзи-якоря: 🔴 высокий приоритет, 🟡 средний, 🔒 занятой слот,

разобраться. Надстрочная двойка (`²`) — счётчик переносов.
