# Архитектура

## Обзор

Moirai — это четыре уровня, общающиеся между собой только через файлы
в одной Google Drive-папке.

```
Пользователь (Telegram)
      ↕  сообщения и ответы
Telegram-бот (VPS, Python/aiogram)
      ↕  чтение и запись через Google Drive API
Хранилище (Google Drive, shared folder)
      ↕  чтение задач, запись артефактов через Claude GDrive connector
Claude Code Routines (облако Anthropic, запуск по cron)
```

Бот и routines никогда не общаются напрямую — ни вебхуков, ни HTTP-
триггеров. Это даёт устойчивость: если одна сторона упала, вторая
продолжает работать, и восстановление происходит автоматически при
подъёме.

## Компоненты

### Пользователь

Пишет в бот свободным текстом. Claude парсит естественный язык
(`завтра`, `асап`, `до пятницы`, теги `#работа`, важность `!важно`)
самостоятельно; бот не занимается NLP.

### Telegram-бот

Python 3.12 + aiogram 3.x, упакован в Docker, работает на Hetzner
в режиме long polling. Две основные задачи:

1. **Приём входящих.** Определяет тип записи (`TASK`, `DONE`, `EVENT`,
   `SLOT`, `NOTE`) по команде/кнопке/простой эвристике, приписывает
   таймстамп, дописывает строку в `inbox.md` на Drive.
2. **Доставка артефактов.** Раз в `OUTPUTS_POLL_INTERVAL_SEC` проверяет
   `outputs/`. При обновлении файла — рендерит компактный Telegram-вид
   и шлёт пользователю.

Плюс локальная обработка команд (`/done`, `/plan`, `/remind` и т.д.)
без обращения к Claude, и таймер напоминаний — проверка `schedule.md`
раз в минуту.

### Хранилище

Одна папка в Google Drive. Бот работает с ней через service account,
Claude — через Google Drive коннектор от имени пользователя. Один и тот
же физический каталог, две разные авторизации, расшарен вручную.

### Claude Code Routines

Облачные агенты от Anthropic, запускаются по расписанию cron. Каждый —
отдельный routine со своим промптом, доступом к той же папке и моделью
Claude. Пять штук: `daily_plan`, `priorities`, `weekly_review`,
`schedule_refresh`. Архивация старых записей — в рамках
`weekly_review`.

## Файлы в Google Drive

```
/TelegramPlanner/
├── inbox.md              # append-only лог от пользователя
├── outputs/
│   ├── daily_plan.md     # таймлайн дня, 08:00
│   ├── weekly_review.md  # обзор недели, Вс 20:00
│   ├── priorities.md     # Eisenhower-ранжирование, 21:00
│   └── schedule.md       # нормализованные даты для таймера бота
└── archive/
    └── YYYY-W##.md       # отработанные записи по неделям
```

Детальные форматы — в [DATA_FORMATS.md](DATA_FORMATS.md).

## Расписание routines

| Routine          | Периодичность           | Что делает                               |
|------------------|-------------------------|------------------------------------------|
| daily_plan       | каждый день 08:00       | таймлайн дня + приоритеты                |
| schedule_refresh | каждый час 09:00–22:00  | обновляет `schedule.md`                  |
| priorities       | каждый день 21:00       | Eisenhower + ранжирование бэклога        |
| weekly_review    | воскресенье 20:00       | обзор недели + чистка inbox в archive    |

## Авторизация

**Бот → Google Drive.** Google service account, создаётся в Google Cloud
Console. JSON-ключ скачивается, кладётся на VPS, монтируется в контейнер
read-only. Email аккаунта (вида `x-y@project.iam.gserviceaccount.com`)
добавляется в sharing настройках папки `/TelegramPlanner/` как Editor.

**Claude → Google Drive.** Через родной коннектор Claude, OAuth от имени
пользователя. Настраивается один раз в интерфейсе Claude.

**Бот → Telegram.** HTTP Bot API, token из BotFather. Long polling,
входящих портов открывать не надо.

**Бот → пользователь.** Whitelist `TELEGRAM_ALLOWED_USER_IDS` в `.env`,
через запятую. Сообщения от чужих игнорируются молча.

## Почему именно так

**Routines только по расписанию, без вебхуков.** Бот и routines получаются
полностью независимыми. Цена — задержка между записью в inbox и
появлением артефакта (максимум час для `schedule_refresh`, несколько
часов для остальных). Для планировщика дел это приемлемо.

**Markdown-файлы, а не база данных.** Claude Drive connector умеет их
читать и писать без промежуточных слоёв. Human-readable — можно открыть
в UI Google Drive и поправить вручную. Минус — отсутствие транзакций;
для single-writer сценария не проблема.

**Google Drive, а не Notion/SQLite/Sheets.** Плоская структура, оба
участника работают с ней изкоробки, бесплатно, без rate limits на этом
объёме.

**VPS, а не PaaS.** Полный контроль, возможность докрутить любые
дополнительные сервисы на том же сервере, фиксированная стоимость,
нет cold starts, нет surprise-биллинга.
