# Инструкции для Claude Code

Этот файл читается автоматически при старте сессии Claude Code
в репозитории. Здесь — общие правила проекта, чтобы не повторять
их в каждом промпте.

## Контекст

Moirai — Telegram-планировщик, обсчитываемый Claude Code Routines.

- Архитектура целиком: [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
- Форматы файлов: [docs/DATA_FORMATS.md](docs/DATA_FORMATS.md).
- Что уже сделано и что следующее: [docs/ROADMAP.md](docs/ROADMAP.md).

Если в промпте не указан нужный контекст — сначала читай эти три
документа, потом приступай к задаче.

## Границы изменений

- `docs/` — источник истины, оркестратор-чат держит их вручную. Не
  редактируй без явной просьбы в промпте.
- `bot/`, `infra/`, будущие `tests/` — рабочая зона.
- `.env` на сервере — не трогать. Все переменные окружения добавляются
  только в `.env.example` в репозитории.

## Рабочие правила

**Один коммит — одна законченная подзадача.** Не сваливай
инфраструктуру и фичу в один коммит.

**Conventional commits.** `feat:`, `fix:`, `docs:`, `chore:`,
`refactor:`, `test:`. Сообщения по-русски или по-английски — но
не мешай стили в рамках одного проекта.

**Стиль Python.** Python 3.12, type hints везде где есть возвращаемое
значение или непустые аргументы. Async — через `asyncio` и aiogram.
Ruff для форматирования и линта (добавим в `pyproject.toml` к Фазе 3).

**Структура модулей.** Один модуль — одна ответственность. Не тащи
в `main.py` и обработку команд, и работу с Drive, и таймер — каждое
в отдельный файл.

**Тесты.** Для нетривиальной логики (парсинг типов, таймер,
рендеринг Telegram-вида) — pytest. Инфраструктурный код не покрываем.

## Что НЕ делать

- Не устанавливать глобально пакеты — всё через `pyproject.toml`.
- Не коммитить секреты. Перед `git add` всегда проверять `git status`.
- Не добавлять зависимости «на будущее» — только под текущую задачу.
- Не реорганизовывать структуру каталогов без явной просьбы.
- Не брать задачи из следующих фаз роадмапа, если текущая не закрыта.

## Полезные команды

Собрать образ бота локально:

```
cd bot && docker build -t moirai-bot .
```

Запустить стек:

```
cd infra && docker compose up -d --build
```

Логи бота:

```
cd infra && docker compose logs -f bot
```

## Отчётность

Оркестратор-чат отчёты читает вручную из вывода сессии — отдельно
в `docs/reports/` их класть не нужно.
